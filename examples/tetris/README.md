# examples/tetris

A minimal Samuel project used as the manual-testing fixture across the
v2.0 release-candidate cycle. Pre-rc.12 manual tests lived in a sibling
directory; this version brings it into the repo so each contributor
has the same starting state.

## Layout

| File / dir | Purpose | Tracked? |
|---|---|---|
| `samuel.toml` | Project config — methodology, guardrails, registries | ✅ |
| `AGENTS.md` | Canonical agent-agnostic context | ✅ |
| `.samuel/tasks/0001-prd-tetris-mvp.md` | Sample PRD with inline `### N.M Title` task headings | ✅ |
| `README.md` | This file | ✅ |
| `CLAUDE.md` | Auto-generated by the built-in Claude translator from `AGENTS.md` | ❌ regenerable |
| `samuel.lock` | Install state for plugins | ❌ regenerable |
| `.samuel/plugins/` | Installed plugin artifacts | ❌ regenerable |
| `.samuel/builtins/` | Framework built-in skills synced from `~/.samuel/builtins/` | ❌ regenerable |
| `.samuel/run/` | Autonomous-loop state (`prd.toon`, `progress.md`) | ❌ regenerable |

Everything in the "regenerable" rows is excluded by the repo's root
`.gitignore` under `examples/*/`.

## Reproducing the manual test sweep

The post-rc.12 validation battery exercises 5 subsystems. Each block
takes under a minute.

```bash
cd examples/tetris
samuel doctor                                            # framework + plugin health, stub advisory
samuel sync                                              # AGENTS.md (root + per-folder) + CLAUDE.md mirror
samuel search go                                         # registry resolution
samuel install actix-web                                 # full install pipeline, no .git/ leak
samuel update actix-web                                  # signature reason printed
samuel update actix-web --allow-unsigned                 # different reason, confirms cache key
samuel run init --prd .samuel/tasks/0001-prd-tetris-mvp.md
samuel run tasks                                         # 5 tasks parsed from inline headings
samuel run start --dry-run --iterations 1 --yes          # one iteration against task 1.1
samuel run done 1.1 --commit-sha "$(git rev-parse HEAD)" --iteration 1
samuel run skip 1.4 --reason "out of scope for fixture"
samuel run reset 1.1
samuel run enqueue "ghost piece (out-of-scope, but enqueueable)"
```

## Why a sample PRD?

The inline `### N.M Title` heading format is the documented happy
path for `samuel run init --prd`, but pre-rc.12 the converter only
recognized a v1-style companion `tasks-*.md` checklist file. Shipping
a real PRD with the supported shape both validates the parser on every
local checkout and gives new users a starting template.

The Tetris MVP itself is just thematic — none of the manual tests
actually build the game. Treat the task descriptions as documentation
fodder, not work items.
