# core/sync.go

**Not skill-sync.** This walks a project tree and generates auto-managed `CLAUDE.md` + `AGENTS.md` files in every non-skipped subdirectory.

## What it does

For each directory:

1. Skip if name in skip-set (`node_modules`, `vendor`, `target`, `dist`, `.git`, `.claude`, …) or starts with `.`.
2. `AnalyzeFolder`: count files by extension, detect tests, identify key files (main.go, package.json, Dockerfile, …).
3. Infer purpose from folder name (`cmd` = "Application entry points", `internal` = "Private packages", `models` = "Data models", …).
4. Render markdown stub: Purpose, Languages, Key Files, Testing.
5. Write to `<dir>/CLAUDE.md` and `<dir>/AGENTS.md` (mirror).

## User-customization preservation

The autogen marker:

```
<!-- Auto-generated by Samuel
```

- If the file has the marker → overwrite freely.
- If the file lacks the marker → assume user-edited, skip unless `--force`.

This is the convention that makes "regenerate" safe.

## Configuration

`SyncOptions`:
- `RootDir` — project root
- `MaxDepth` — `-1` for unlimited
- `Force` — overwrite user-edited files
- `DryRun` — preview without writing

`SyncResult`:
- `Created`, `Updated`, `Skipped`, `Errors`

## Maps it carries

- `extensionLanguageMap` — 35+ entries, includes `.toml`.
- `folderPurposeMap` — 35 conventional names with descriptions.
- `keyFileDescriptions` — 17 well-known entry-point / manifest filenames.
- `skipDirSet` — 25 common build/cache/vendor dirs to skip.

## v2 implications #rescue

Keep this whole thing. It's a clean self-contained feature:

- No dependency on the registry.
- No dependency on the install path.
- Purely walks + analyzes + writes.

Refactor opportunities:

- Make the maps configurable per project (via `samuel.toml` overrides) so users can adjust folder purposes or skip rules without forking.
- Drive language detection from installed plugins instead of a hardcoded map.
- Optional: emit a single `design.md` per folder in addition to CLAUDE.md/AGENTS.md once the user's [[concepts/extensibility-design]] settles.

In v2, this becomes a built-in core capability — `samuel sync` regenerates per-folder context files. It belongs in the framework, not a plugin, because the autogen marker convention is part of the Samuel Way.
