## Key claims

### Root setup (`root.go`)

- Cobra. `Use: "samuel"`. `SilenceUsage`, `SilenceErrors`.
- `SuggestionsMinimumDistance: 2` ‚Äî "did you mean?" hint for typos. Same default as cargo and gh.
- **Persistent global flags**: `-v / --verbose`, `--no-color`, `--json`, `--no-deprecation`.
- `JSONMode(cmd)` is the helper every command uses to branch its output formatter.
- Build-time vars: `Version`, `Commit`, `BuildDate`.

### `samuel run` (was `samuel auto`)

**v1 already renamed.** From `auto.go:13-19`:

> "runCmd is the v3.0.0 primary verb for the autonomous Ralph Wiggum loop. `samuel auto` is preserved as a permanent top-level alias (not Hidden, not deprecated) so muscle memory and existing community references keep working forever."

Implication: v2's [[synthesis/auto-mode-v2-design]] rename was already aligned with v1's direction ‚Äî but we go further (multi-methodology under `samuel run [methodology]`).

### Smart bare invocation #pattern

`samuel run` with no args (`runRunBare`):

- If `.claude/auto/prd.json` exists ‚Üí show status (read-only).
- If missing ‚Üí print actionable help to stderr, exit non-zero.

**Never silently starts a loop.** Comment: "preventing the v2-era footgun where a stray `samuel auto` could kick off pilot mode in a directory the user wasn't expecting." Strong UX pattern.

### Legacy/deprecation pattern (`legacy.go`)

```go
func redirectAndRun(newPath string, handler func(*cobra.Command, []string) error) func(...) error {
    return func(cmd, args) error {
        if !deprecationSuppressed(cmd) {
            oldPath := cmd.CommandPath()
            fmt.Fprintf(os.Stderr, "samuel: %q was renamed in v3.0.0. Use: %s\n", oldPath, newPath)
            fmt.Fprintln(os.Stderr, "        Set SAMUEL_NO_DEPRECATION=1 (or --no-deprecation) to silence. Removes in v3.1.0.")
        }
        return handler(cmd, args)
    }
}
```

- Wraps a handler with one-line stderr deprecation note.
- Suppressible via `SAMUEL_NO_DEPRECATION=1` env or `--no-deprecation` flag.
- Stderr-only ‚Äî JSON consumers reading stdout are unaffected.
- Hidden in `--help` but functional. Scheduled removal in v3.1.0.

The pattern is small, elegant, and respects scripting use cases.

### JSON mode (`JSONMode(cmd)`)

Every command checks `JSONMode(cmd)` and branches to `ui.PrintJSON(...)` if set. Schema-versioned envelope from [[entities/ui-package]].

`json_helpers.go` adds `PrintJSONForCmd` / `PrintJSONErrorForCmd` that pull the **invoked** command path (not the handler's hardcoded label) for legacy-alias telemetry.

### `samuel init` (`init.go`)

The most complex command. Flow:

```
parseInitFlags()
validateInitTarget()        # refuses to init inside the Samuel source repo
selectComponents()           # interactive prompts or flags
displayAndConfirm()
downloadFramework()          # github.Client ‚Üí cache
installAndSetup()            # extract template/ ‚Üí project
runOrchestratorBundle()      # v4: install gstack + gbrain + samuel-skills
saveInitConfig()             # write samuel.yaml
```

Flags:
- `--template {full,starter,minimal}`, `--languages a,b,c`, `--frameworks a,b,c`
- `--force`, `--non-interactive`
- v4 orchestrator escape hatches: `--skip-gstack`, `--skip-gbrain`, `--gbrain-binary <path>`, `--no-symlink`, `--no-orchestrator` (drop entirely; legacy v3 init only)

Language alias expansion (`init.go:88-116`): `ts/js/javascript ‚Üí typescript`, `py ‚Üí python`, `cs ‚Üí csharp`, `c++/c ‚Üí cpp`, `rb ‚Üí ruby`, `sh/bash ‚Üí shell`. Same for frameworks (`next ‚Üí nextjs`, `spring ‚Üí spring-boot-java`).

Samuel-repo detection (`init.go:140-160`): looks for `template/CLAUDE.md` + `packages/cli/go.mod` markers. Refuses to init inside Samuel itself.

### `samuel doctor`

Unified health output:

```
- config file present + valid
- CLAUDE.md present
- AGENTS.md present
- directory structure correct
- installed components exist
- skills validate
- if .claude/auto/ exists: auto-mode health
- if config: local-modification check
- if !--no-orchestrator: gstack + gbrain + samuel-skills HealthStatus from [[entities/orchestrator]]
```

`--fix` mode: auto-create missing dirs + re-download missing components from cache. Re-runs after fix to verify.

### `samuel add` flexibility

Accepts both arg orders:
- v3 (preferred): `samuel add react` (inferred), `samuel add react framework` (name first)
- v2 (still works): `samuel add framework react` (type first)

Type aliases:
- `language | lang | l`
- `framework | fw | f`
- `workflow | wf | w`

If type is omitted, calls `core.InferComponentType(name)` (see [[entities/registry]]) ‚Äî resolves uniquely or returns an ambiguity error.

### `samuel skill` matches the standard

Four subcommands map 1:1 to the Agent Skills lifecycle:
- `create <name>` ‚Äî scaffold (SKILL.md template + scripts/references/assets dirs with .gitkeeps).
- `validate [name]` ‚Äî run [[entities/skill-md]] validation. All or one.
- `list` ‚Äî scan `.claude/skills/`, show name + description + valid flag.
- `info <name>` ‚Äî full metadata + body line count.

All four have JSON mode with structured output.

### Other patterns worth noting

- **Spinner during downloads** ([[entities/ui-package]] used). `samuel add` shows `Downloading <name>...` spinner that resolves to ‚úì or ‚úó.
- **Helpful error messages**. `samuel add unknown-lang` returns `unknown language: unknown-lang\nRun 'samuel list --available --type languages' to see available languages.`
- **Pre-flight checks**. Most commands fail-fast if Samuel isn't initialized: `no Samuel installation found. Run 'samuel init' first`.

## Assessment

- **Credibility**: high.
- **Quality**: very polished CLI. Strong UX patterns (smart bare invocation, deprecation cleanness, JSON mode everywhere, type inference, helpful error messages).
- **Surface size**: large. ~20 top-level commands + nested subcommands. v2 should trim aggressively given the "thin core" decision.

## v2 implications

### `#rescue` (patterns)

- Smart bare invocation for `samuel run`. Same shape, same anti-footgun.
- `redirectAndRun` deprecation pattern. (Not used in v2.0 clean-break, but worth keeping the utility ‚Äî future renames will need it.)
- `JSONMode(cmd)` + persistent `--json` flag. Every command emits JSON.
- Type-inference for argument shorthand.
- Spinner during slow operations + ‚úì/‚úó resolution.
- Helpful error messages with "run X to learn more" hints.
- `samuel doctor` unified health view.
- `samuel skill` subcommand structure (port verbatim ‚Äî Agent Skills standard hasn't changed).
- Admin/main split: ops under `admin`, primary verbs at root.

### `#refactor`

- **`add`/`remove` ‚Üí `install`/`uninstall`** with plugin terminology. Drop the language/framework/workflow distinction per [[concepts/extensibility-design]].
- **Language alias expansion** becomes plugin-discovery: `samuel install go-guide` resolves directly; `samuel install go` searches for it.
- **`samuel init`** becomes much smaller: no gstack/gbrain, no language/framework selection (those are plugins). Maybe just: directory layout + `samuel.toml` skeleton + Samuel-self-install (the SamuelComponent equivalent).
- **`samuel run` adds methodology positional**: `samuel run [methodology]`, default from `samuel.toml`. Subcommands stay.

### `#drop`

- Legacy top-level `config` and `sync` aliases (clean break).
- `samuel admin diff` against versions (v2 doesn't have a single-repo version model).
- `samuel run task list/complete/skip/reset` deprecated nested forms.
- `--skip-gstack` / `--skip-gbrain` / `--gbrain-binary` / `--no-orchestrator` flags on init.
- The `samuel list` vs `samuel ls` duplication ‚Äî pick `ls`.
