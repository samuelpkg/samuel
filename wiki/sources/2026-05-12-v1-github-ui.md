# v1 GitHub + UI

Ingest pass 5. Two small utility packages.

## Files

- `samuel_v1/internal/github/client.go` (295 lines) — `Client`, `Release`, `Tag`, `VersionInfo`, archive/file downloads
- `samuel_v1/internal/ui/output.go` (138) — colored output + symbols
- `samuel_v1/internal/ui/json.go` (57) — JSON envelope + schema versioning
- `samuel_v1/internal/ui/prompts.go` (182) — Select, MultiSelect, Confirm, Input via promptui
- `samuel_v1/internal/ui/spinner.go` (108) — spinner + progress bar via schollz/progressbar

## Key claims — GitHub client

- Constants for URL templates: archive tar.gz, branch archive, latest release, tags. (`client.go:11-34`)
- `Client{httpClient (Timeout: 30s), owner, repo}`. (`client.go:41-56`)
- Every request sets `User-Agent: samuel-cli` + `Accept: application/vnd.github.v3+json` for API calls.
- `MaxDownloadFileSize = 10 MB` per file (cf. 100 MB in [[entities/downloader-extractor]] which guards extracted size, not download size). (`client.go:36-38`)
- `GetLatestVersionOrBranch` falls back to (`"dev"`, true) when no releases exist — drives the v3-era branch-archive flow.
- `DownloadFile(version, path)` uses `raw.githubusercontent.com` for single-file fetches. Limit-reader prevents OOM.
- `CheckForUpdates(current)` returns `{Current, Latest, UpdateNeeded, ReleaseNotes}` — drives `samuel update` and version comparison.
- v-prefix stripping is duplicated in two places (`GetLatestVersionOrBranch`, `CheckForUpdates`) — minor refactor opportunity.

## Key claims — UI package

### `output.go` — colored text

- Six color tags (success/error/warn/info/bold/dim) via fatih/color.
- Six symbols: `✓ ✗ ⚠ → ○ ●` (success/error/warn/info/pending/active).
- Function families: `Success/Error/Warn/Info/Print/Bold/Dim/Header/Section/ListItem` + indented `*Item` variants + `TableRow` / `ColoredTableRow`.
- `DisableColors()` toggles via `color.NoColor` — for CI / piped output.
- Error variants write to **stderr**; everything else to stdout. Consistent.

### `json.go` — versioned envelope

```json
{
  "schemaVersion": 3,
  "command": "<invoked command path>",
  "success": true,
  "data": { ... },
  "error": "..."
}
```

- `JSONSchemaVersion = 3` is current. Constant carries a comment explaining what changed from v2 (added `schemaVersion` field, `command` now reflects invoked path not handler label). (`json.go:9-19`)
- `PrintJSON(cmd, data)` → stdout (indented 2 spaces).
- `PrintJSONError(cmd, err)` → stderr.
- **This versioned envelope is the cleanest patten in the UI package**. Lets consumers pin to a schema and branch on field presence.

### `prompts.go` — interactive prompts (promptui)

- `Select(label, options)` — single-choice. ▸ cyan active + ✓ green selected. Size 10.
- `MultiSelect(label, options, defaults)` — **custom implementation** since promptui doesn't have native multi-select. Loops a Select with `[✓]/[ ]` toggle prefixes + a "Done" sentinel. Works but ugly.
- `Confirm(label, defaultYes)` — Y/N prompt. Honors Ctrl+C / abort.
- `Input(label, default, validate)` — free-text with validator function.
- `InputWithPlaceholder(label, placeholder)` — prints `(e.g., <placeholder>)` above the prompt.

### `spinner.go` — schollz/progressbar v3

- `Spinner` — indeterminate. 100ms tick. `sync.Once` guards `Stop` against double-close.
- `ProgressBar` — determinate. Green saucer theme.
- Spinner has `Success(msg)` / `Error(msg)` helpers that stop and print a status line in one call.

## v2 implications

### GitHub client — `#refactor`

Drop the v1 model of "one client for the Samuel repo" and generalize.

- Keep the **shape**: `Client{httpClient, owner, repo}`, 30s timeout, User-Agent, size cap, limit-reader.
- Generalize: every plugin install creates a `github.Client` for its source repo. v2's plugin loader is the caller; no global Samuel-repo coupling.
- Extend with: `Authorization: Bearer <token>` for private repos (env: `SAMUEL_GITHUB_TOKEN` or `GH_TOKEN`).
- Extend with: Sigstore signature fetch (`<release>/<asset>.sig`, `.crt`) for [[concepts/versioning-compatibility]].
- For OCI plugins, this gets a sibling `oci.Client` using the OCI distribution-spec (or a library like `oras-project/oras-go`).

### UI package — mixed

- **JSON envelope** — `#rescue`. Port verbatim. Bump `JSONSchemaVersion` to 4 if v2 changes shape. The pinned-version pattern is the right answer.
- **Output colors + symbols** — `#rescue` shape, `#refactor` library. Charmbracelet/lipgloss is the 2026 standard. Same six categories.
- **Prompts** — `#refactor` library. promptui works but is showing its age; the custom MultiSelect is awkward. Charmbracelet/huh has native multi-select, better keyboard handling, and a much nicer programming model. Same API surface from the caller's perspective; swap the implementation.
- **Spinner / progress** — `#refactor` library. Bubbles (`bubbles/spinner`, `bubbles/progress`) integrates cleanly with Bubble Tea if v2 ever wants a richer TUI. For the simple "spin while running" use case schollz/progressbar is fine.

### Recommended v2 stack #v2-decision

- **Color**: `charmbracelet/lipgloss` — theme tokens, terminal capability detection.
- **Prompts**: `charmbracelet/huh` — Select, MultiSelect, Confirm, Input, validation, form-style chaining.
- **Spinner**: `charmbracelet/bubbles/spinner` for short-lived; `bubbletea` model only if we build a real TUI (samuel doctor with live updates, samuel run with iteration tracker).
- **HTTP**: keep `net/http` + custom Client. No need for a framework.

Same abstractions, modern libraries.
